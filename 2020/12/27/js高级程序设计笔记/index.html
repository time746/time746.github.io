垃圾收集
标记清除
记住垃圾回收只作用于对象。
javascript最常用的垃圾收集方式。当变量进入环境时，这个变量标记为“进入环境”；而当变量离开环境时，则将其标记为“离开环境”。可以使用一个“进入环境”的变量列表及一个“离开环境”的变量列表来跟踪变量的变化，也可以翻转某个特殊的位来记录一个变量何时进入环境及离开环境。

标记清除也可以 这样: 遍历所有栈中的变量, 为root , 这样就能知道哪些对象被引用了, 被引用的对象添加上标记,遍历完成后, 没有标记的就可以清除了

当执行到一段可执行代码时(全局代码就是全局环境, 函数就是函数环境), 会创建一个执行环境

作用域链, 作用域链在函数声明好的时候就确定了, 但是没有创建

执行环境中会有两个重要的属性, variable object变量对象和 作用域链
对于全局环境来说, 变量对象就是window
对于函数环境来说 变量对象称为活动对象

活动对象active object: 里面包含this, arguments, 和其他参数

当执行到一个函数时, 会创建一个执行环境 execution context 和 作用域链
这个作用域链会赋值给函数的特殊属性, [[scope]]
然后使用this, arguments, 和其他函数接收的参数初始化活动对象active object
在作用域链中 外部函数的活动对象处于第二位

总的来说, 作用域链理解为一个数组, 里面有上层作用域的变量对象,
scope[0] 保存的是上一层的变量对象, scope[scope.length-1] 保存的是global变量对象
函数创建时会创建一个执行环境, 里面有作用域链,, 当活动对初始化好后会被添加进执行环境中
